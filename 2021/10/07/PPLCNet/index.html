PPLCNet

这是百度团队针对Intel-CPU端加速而设计的高推理速度和高性能的PP-LCNet，原论文链接

摘要

We propose a lightweight CPU network based on the MKLDNN acceleration strategy, named PP-LCNet, which improves the performance of lightweight models on multiple tasks. This paper lists technologies which can improve network accuracy while the latency is almost constant. With these improvements, the accuracy of PP-LCNet can greatly surpass the previous network structure with the same inference time for classification. As shown in Figure 1, it outperforms the most state-of-the-art models. And for downstream tasks of computer vision, it also performs very well, such as object detection, semantic segmentation, etc. All our experiments are implemented based on PaddlePaddle1. Code and pretrained models are available at PaddleClas2.

介绍

提出了基于MKLDNN加速的轻量CPU模型，列举了一些提升精度延迟不变的技术，在图像分类任务，PP-LCNet在推理延迟、精度方面优于ShuffleNetV2、MobileNetV3等。

方法

由于百度和Intel合作的关系：），这次百度的PPLCNet考虑了Intel CPU的推理速度，推出了这个网络。

  • Better activation function

    采用了H-Swish激活函数代替了ReLU，提升性能，推理速度不变

    在移动设备上计算sigmoid函数的代价很大，所以使用了ReLU6版本的h-swish激活函数

  • Larger convolution kernels

    由于不同尺寸的卷积对网络性能的影响会降低模型的推理速度，作者实验发现在网络的尾部采用$5\times5$的卷积核可以取得相近的效果。

    上表展示了在尾部使用大卷积有更好的效果。

  • Larger dimensional $1\times1$conv layer after GAP

    由于GAP后输出维度小，直接分类的话性能小，所以添加了一个1280维度的$1\times1$的卷积，在小的推理延迟下提高性能。

  • SE modules

    虽然SE被广泛应用到网络架构中，但在Intel CPU端，耗时会提高所以不能在整个网络中使用，

    论文中说，在把SE模块放在模型的尾部有更好的作用，SE模块的两个激活函数为ReLU和H-Sigmoid。

    However, on Intel CPUs, the SE module increases the inference time, so that we cannot
    use it for the whole network. In fact, we have done a lot of experiments and observed that when the SE module is located at the end of the network, it can play a better role.
    So we just add the SE module to the blocks near the tail of the network. This results in a better accuracy-speed balance. As with MobileNetV3, the activation functions for the two layers of the SE module are ReLU and H-Sigmoid respectively.

    下图对比了SE的位置对模型性能的影响，可以发现仅在尾部添加两个SE模块效果更好。

总结

上面两个表展示了移除不同模块的影响，可以总结

  • 使用dropout提高了精度
  • 在GAP后采用更大FC层提高了模型性能
  • 在合适的位置使用少量的SE
  • 使用大的卷积核提高性能，推理耗时几乎不变
  • 使用H-Swish激活函数
